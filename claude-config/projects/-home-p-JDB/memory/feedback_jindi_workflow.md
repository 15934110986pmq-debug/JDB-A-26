**Why**: 这些偏好在 Q2/Q3/Q4 的连续工作中被多次确认，已成项目级范式；不要在每次新章节里重新发明。

**How to apply**: 写新章节 / 修改现有章节时，自动遵循；写代码时优先把这些机制嵌入。

## 1. 红队多 AI 验证作为外部审查机制

每完成一问就要：
- 写 `Q{x}_审查清单.md`（提交给独立 AI 用）
- 用户去贴给 3-5 个独立 AI（ChatGPT/Gemini/独立 Claude/DeepSeek/豆包），收集回复
- 写 `Q{x}_三审综合.md`（投票表 + 共识 + 分歧 + P0/P1/P2 清单）
- 按 P0 修正代码 + 论文

清单格式固定：§0 项目背景 + §1 模型框架 + §2 实证数据 + §3 逐条问题 + §4 5 个必填 yes/no + §5 防误解提醒 + §6 强制返回格式。

## 2. 诚实声明范式（"统计 vs 物理"）

任何统计检验结论必须明确区分：
- **统计判断**："在 X% 显著性水平下不拒绝 H0"
- **物理事实**："不等于 X 严格为 0"

例：Q3 系统偏差判定不能写"不存在系统偏差"，要写"统计上无显著证据 + 工程语境补充段（噪声稀释 / 信噪比 < 0.05 / 解析需 64× 样本量）"。

## 3. J\* 噪声底诊断（杀手锏）

LS 真解处 $J^* \approx \sigma_x^2 + \sigma_y^2$；过拟合伪解 $J^* < \sigma_x^2 + \sigma_y^2$ 是反向铁证。
- Q2 用此识破原 NM 早停（C4 J\*=3.139 真值 1.827）
- Q4 用此识破 d=5.02 边界候选过拟合
- 以后类似场景**优先用噪声底比对** 判定真伪解

## 4. 双解对照（标称 + 鲁棒）

Q4 建立的范式：
- **标称解**：严格按题面约束，等权 max ∑ x_i
- **鲁棒解**：chance constraint $z=1.645$ + ε=0.1s 过渡时间
- 论文同时报告两版 + 对照表（公共 / 仅标称 / 仅鲁棒）

意外发现："鲁棒解反多 1 任务"——ILP 全局重组在更严约束下可能找到更安全候选。

## 5. 段内密集候选 + ILP（vs 段内取 d_min 一个）

旧做法：每个可行段只取 d 最小一点为执行时刻 → 候选数少，ILP 选择空间小。
新做法：段内按 (T_aim+ε) 间隔放多个候选 → 让 ILP 自由选最优组合。

## 6. K 拐点选择

Q4 射击同目标最大次数 K：
- K=1: 任务覆盖率太低（旧 unique 假设）
- K=∞: 同目标重复过度（S12 25 次物理不合理）
- **K=3 是命中率累积 99.66% 拐点**（边际收益 < 2%）

类似 trade-off 出现时优先找命中率拐点 / 边际收益拐点作为参数选择论证。

## 7. 8 章论文骨架（参考 PDF）

最终成稿用国奖标准 8 章式（不是 Qx.md 自包含 9 节）。各 Qx.md 是工作草稿，合稿时按 `CONVENTIONS.md §3.3` 拆分。

## 8. ILP 用 scipy.milp（非 PuLP）

venv 中 PuLP 未装，但 scipy.milp + HiGHS 已就绪。Q4 调度直接用 `from scipy.optimize import milp, LinearConstraint, Bounds`。

## 9. 不动其他章节的代码 / 改动隔离

修 Q3 时发现 q_utils.py 的 `feasible_domain` 在 Q3 数据下符号反；选择**在 q3_solve 内 inline `feasible_domain_correct`**，不动 q_utils.py（避免破坏 Q2 工作）。

## 10. 用户消息特征

- 极度简短（"P0"、"push 一下"、"改答案"、"开始 Q3?"等单句）
- 偏好动作而非长讨论
- 多轮分歧时倾向"按数据 / 多数派"决策
- 对"被绕晕"敏感 → 解释要简明，避免堆砌术语
