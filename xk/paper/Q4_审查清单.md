## 1. 模型框架与算法（请逐项核对）

### 1.1 候选任务生成（滑动窗）

对每个目标 $g\in\{\text{S01..S18, P01..P18}\}$ 与每个时刻 $j$：

1. 计算瞬时距离 $d_j = \|\mathbf{p}_j - \mathbf{p}_g\|$
2. 检查瞬时约束：$d\in[d_{\min},d_{\max}]$, $|v|\le v_{\max}$, $|a|\le a_{\max}$
3. **滑动窗判定**：以 $j$ 为准备开始时刻，$[j, j+W)$ 内**全部**满足上述约束（$W=15$ 射击 / $W=5$ 拍照）
4. 找连续可行段，**段内取距离最小点为执行时刻** $j_{\text{exec}} = \arg\min_{k\in\text{段}} d_k$
5. 准备开始时刻 = $t_{\text{exec}} - T_{\text{aim}}$

### 1.2 拍照同目标多角度去重

对每个拍照目标，按 $j_{\text{exec}}$ 时刻顺序遍历候选；保留视角与已选候选差 ≥ 60° 的：

$$\text{view}(\mathbf p, \mathbf p_g) = \mathrm{atan2}(y - y_g,\ x - x_g),\quad |\Delta\text{view}|\ge 60°$$

### 1.3 ILP 调度

```
变量: x_i ∈ {0, 1}  (i = 1..n=59 候选)
目标: max ∑ x_i

约束:
  (a) 时段不重叠: 对每对 (i, j) i<j, 若 t_exec_i > t_prep_j 则 x_i + x_j ≤ 1
  (b) 射击唯一: 同一射击目标的多个候选 ∑_{k} x_k ≤ 1
                (拍照同目标允许多角度, 不加该约束)

求解: scipy.optimize.milp (HiGHS solver), integrality=ones, bounds=[0,1]
```

## 3. 请独立 AI 逐条作答

### A. 模型构建合理性

A1. **滑动窗判定**：约束"校准期间持续满足"被建模为"窗内全部点满足"。
- (i) 这是否等价于题面"持续满足"？
- (ii) 由于离散采样 dt=0.1s，窗端点 ε 内的违反不能检测——是否需要在窗内做线性插值？

A2. **段内取距离最小点为执行时刻**：
- (i) 在保证窗内全程满足约束的前提下，取 $d_{\min}$ 是否最大化命中率？
- (ii) 是否应改取 $|v|$ 最小或 $|a|$ 最小（更稳态）？给出三选一的论证。

A3. **拍照视角定义**：$\text{view} = \mathrm{atan2}(y_{\mathrm{robot}} - y_g, x_{\mathrm{robot}} - x_g)$ 是从目标看机器人的方向。题面"不同角度间方向角差异"是否就是这个角度？还是应该改为机器人朝向（从机器人看目标）？两种定义差 180° 但 60° 阈值下结果可能差异很小。

### B. 调度算法

B1. **ILP 形式**：
- (i) 每对 (i, j) 时段冲突约束 $x_i + x_j \le 1$ 是否覆盖所有冲突情形？是否需要 transitive closure（如 i 与 j 冲突, j 与 k 冲突, i 与 k 也冲突）？
- (ii) 复杂度 O(n²) ≈ 1740 约束（n=59），HiGHS 是否秒级求解？给出实际求解时间感觉。

B2. **射击唯一性约束**：题面"单次命中率 85%"暗示同目标可能需多次射击才命中。
- (i) "目标完成数"应当如何定义？(a) 每次射击算 1 次任务（即使打不中）；(b) 唯一目标算 1 次（无论射击多少次）；(c) 期望命中数（含 0.85 折扣）
- (ii) 当前实现采用 (b)。是否应改为 (a) 让 ILP 选多次射击同目标以提高命中概率？ trade-off 是什么？

B3. **目标函数权重**：当前 max ∑ x_i 等权 1。
- (i) 是否应该按 (射击 × 0.85, 拍照 × 1.0) 加权？这样可能让 ILP 偏好拍照（已经全成功）。
- (ii) 拍照"鼓励多角度"是否应该给同目标多次拍照额外奖励（如递减权重 1.0, 0.5, 0.25, ...）？

### C. 约束完整性

C1. **任务间过渡时间**：题面没明确"任务间是否需要间隔"。当前实现允许 $t_{\mathrm{exec}, i} = t_{\mathrm{prep}, j}$ 紧贴。
- (i) 工程上一个任务结束到下一个任务开始是否需要切换时间（如相机模式切换 0.1s）？
- (ii) 若需要 ε 间隔，应在 ILP 时段约束中加 $t_{\mathrm{exec}, i} + \epsilon \le t_{\mathrm{prep}, j}$。

C2. **机器人能否同时执行多任务**：当前默认互斥。如果机器人是无人车 + 摄像头 + 武器系统，物理上**拍照与射击可同时执行**（只要约束不冲突）。题面是否暗示并发？保守解释：单任务，已采用。

C3. **数据不确定度**：Q3 的 KF 后验位置 σ ≈ 0.61 m。距离 d=5.02m (S06) 的边界值在不确定度下可能违反 $d\ge 5$。
- (i) 是否应该用 $d - 1.96\sigma_d \ge 5$ 做鲁棒约束？
- (ii) σ_d 如何由 σ_x, σ_y 推算（$\sigma_d^2 = (\partial d/\partial x)^2\sigma_x^2 + (\partial d/\partial y)^2\sigma_y^2$）？

C4. **加速度估计**：当前用 KF 速度的中心差分。
- (i) 这是否引入额外噪声？是否应直接用 KF 模型的过程噪声 $\sigma_a = 1.5$ 作为先验上界？
- (ii) 入选任务的 $|a| \le 0.964$ 全部远小于 $\sigma_a = 1.5$ 限制，说明 KF 输出在低速段加速度本就小——约束不主动起作用？

### D. 解的最优性与稳健性

D1. **ILP 给出 20 任务**。验证最优性：
- (i) 是否检查过 ILP solver 收敛状态？
- (ii) 在所有 (n=59 选 20) 中，是否有备选 21 任务但被某个约束阻塞？请给出诊断方法（slack 分析）。

D2. **敏感性分析**（建议补做）：
- (i) 把 d_max（射击 30 → 25）调严，能完成几个任务？
- (ii) 把视角差 60° → 90° 调严，拍照覆盖减少多少？
- (iii) 把 KF σ_a 从 1.5 调到 1.0，是否影响候选生成？

D3. **解的物理验证**：
- (i) S06 在 t=462.2 时 d=5.02m, v=0.592, |a|=0.299——是否物理合理？（机器人 0.59 m/s 慢速接近射击目标 5 m 距离）
- (ii) P10 在 t=481.1 时 v=1.500m/s（边界），KF 速度估计的不确定度若 5%，则真实 v 可能在 [1.43, 1.58]，跨越约束边界——是否应剔除该候选？

## 5. 请避免的常见错误

- **不要**把 28 m/s 当作机器人真实速度——那是有限差分对原始噪声数据的虚假估计；KF 平滑后是 2.15 m/s 中位（约束可行）。
- **不要**用"射击 1 次=1 任务"理解 v=2.0 边界严苛性，因为命中率 85% 暗示工程容差。
- **不要**把"同目标多角度 ≥ 60°"误解为 30°（题面明确 60°）。
- **不要**忽略射击唯一性约束（同目标多次射击不增加完成数）。
- **不要**把 ILP 求解时间作为方法论问题（n=59 在 HiGHS 下 < 1s）。
