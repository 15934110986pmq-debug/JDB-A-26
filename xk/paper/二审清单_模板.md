# 论文章节二审清单（Meta-Review Template）

> **定位**：在一审（红队独立验证，例如 `Q2_审查清单.md`）完成、且作者已根据一审反馈打补丁之后，进行的**二轮审查**。  
> **目的**：审查"修正本身"——补丁可能过强、过弱、互相矛盾、引入新错误，或暴露所有一审者共同的盲点。  
> **价值**：单次评审 + 单次修正不足以稳健；二审是把"评审 → 修补"循环再闭合一次的关键步骤。

---

## 0. 项目背景速递（与一审清单相同，必须放在 prompt 最前）

```
项目：<项目名 / 题目编号>
章节：<被审章节，如 Q3>
论文文件：<相对路径>
仓库链接：<GitHub 直链>

本章核心结论速递（请验证）：
  <主要参数估计 + CI>
  <主要诊断指标 + 阈值>
  <关键模型选择>
```

---

## 1. 审查输入（提供给二审者的材料）

二审者必须收到以下四份材料，缺一不可：

1. **原始论文章节**（被审章节的当前版本，**已包含一审修正**）
2. **一审反馈汇总**：
   - 至少两位独立 AI 评审的完整意见（建议 ChatGPT + Gemini + DeepSeek 各一份）
   - 每位评审的"必须修正"清单
3. **修正 diff**：作者根据一审反馈打的所有补丁（git diff 形式或对照表）
4. **修正决策日志**：哪些建议被采纳、哪些被拒绝、拒绝理由

---

## 2. 二审者的六类任务

### A. 修正完整性核查（patches-coverage check）

> 一审"必须修正"清单中的每一项是否都被修了？有没有遗漏或避重就轻？

| 检查点 | 询问 | 输出 |
|---|---|---|
| A.1 | 把一审所有"必须修正"项列成表，对每项标记 ✓修正 / ✗未修 / ⚠️部分修 | 完整性表 |
| A.2 | 跨多位评审的同一议题，是否被合并处理？还是只采纳了某一位的措辞？ | 是否有信息丢失 |
| A.3 | 评审"建议改进（非必须）"项是否被记录决策（采纳 or 拒绝 + 理由）？ | 决策日志缺口 |

### B. 修正强度核查（over-/under-correction check）

> 每处补丁是否做过头（over-correction，把好东西也删了）或做不到位（under-correction，遗漏关键含义）？

| 检查点 | 询问 | 红旗信号 |
|---|---|---|
| B.1 | 补丁是否把原本正确的描述也删去/弱化？ | 优点列表中性能数值被无故下调 |
| B.2 | 补丁是否仅改了表层措辞而未触及实质问题？ | "略保守"→"略偏高"但未量化超界程度 |
| B.3 | 补丁是否引入未经验证的"诚实区间"或"新数字"？ | 任何加进来的定量声明（区间、阈值、比例）都应有数值或文献支撑 |
| B.4 | 补丁是否给出**单一原因**解读，而实际有多种可能？ | "X 比 Y 小正好反映 Z" — 检查是否还有 W、V 也能解释 |

### C. 修正一致性核查（cross-section consistency check）

> 一处补丁打完后，全篇还有多少地方仍残留旧定性？

| 检查点 | 询问 | 工具 |
|---|---|---|
| C.1 | overview 表格 / 章节首段 / 小结 / 优缺点列表，措辞是否与新结论一致？ | `grep` 关键词扫查 |
| C.2 | 数值核心声明（如 NIS 上界、CI 宽度）是否在所有出现位置一致？ | 关键数字的 grep |
| C.3 | 是否有"双重证据 / 完美吻合 / 完全验证"等强声称残留？ | 强力定性词清单扫查 |
| C.4 | 摘要 / 引言 / 结论里是否仍宣传被推翻的卖点？ | 全章逐节核对 |

### D. 新错引入核查（regression check）

> 补丁本身是否引入新的数学错误、过强声称、未验证估算？

| 检查点 | 询问 | 红旗信号 |
|---|---|---|
| D.1 | 补丁中所有定量声明是否都跑了数值验证？ | "估算约 0.04–0.06" — 跑过 Riccati 没？ |
| D.2 | 补丁中的临界值/分位数是否对应正确分布与样本量？ | $\chi^2$ 单样本 vs 样本均值的不同临界值 |
| D.3 | 补丁中"$X\sigma$ 越界"等显著性声明是否假设了样本独立？ | 时序模型的样本不独立 |
| D.4 | 新加的术语是否与代码实现匹配？ | "非参 Bootstrap" — 看代码是 parametric 还是 non-parametric |
| D.5 | 拓展/类比到下章节的承诺是否兑现可能？ | "Q3 将升级为 X" — Q3 真有空间做吗？ |

### E. 集体盲点扫查（collective-blind-spot sweep）

> 所有一审者都漏掉了什么？这是二审最关键也最难的环节。

| 检查点 | 询问 | 例子 |
|---|---|---|
| E.1 | "教科书数字"是否被无脑套用？ | $\chi^2(2)$ 95% 上界对单样本是 5.99，对 N=15 mean 是 2.99，对 N=6948 mean 是 2.04 —— 三种语境完全不同 |
| E.2 | 同源数据的"自洽性"是否被当成了"独立验证"？ | $J^* = 2(\hat\sigma_x^2+\hat\sigma_y^2)$ 是代数恒等式，不是模型验证 |
| E.3 | 方法名称是否被使用得过于宽泛？ | "Bootstrap" 既可以参数化也可以非参，不同语境下含义截然不同 |
| E.4 | 渐近结果是否被滥用到有限样本场景？ | CRB 渐近成立 ≠ 有限样本下严格 |
| E.5 | "工程合理"是否被混用为"统计严格"？ | 选主解的工程判据 ≠ 假设检验 |

**E 类的实战技巧**：把所有一审者的"通过/同意"项重新审视一遍——大家都同意的最容易是集体盲点。

### F. 数值兜底验证（numerical sanity check）

> 凡补丁中加入新数字、新区间、新阈值，必须用代码或闭式解兜底。

| 检查点 | 工具 |
|---|---|
| F.1 | 概率分位数 → `scipy.stats.<dist>.ppf` |
| F.2 | KF 稳态方差 → `scipy.linalg.solve_discrete_are` |
| F.3 | 不确定度传播 → 蒙特卡洛 1000 次以上 |
| F.4 | 文献引用的数字 → 直接查原文公式 |

口头估算（"我估在 0.04–0.06 区间"）一律要求转成数值或撤回。

## 4. 二审者的角色设定（prompt 模板首段）

```
你是一位资深的论文同行评审，正在进行**二轮审查**。第一轮已由若干独立 AI 评审完成，
作者据此打了补丁。你的任务**不是**重做一审、而是审查"修正本身"。

特别警惕三类问题：
1. 修正过强：把原本正确的卖点也无故弱化
2. 修正不够：仅改措辞而未触及实质
3. 修正引入新错：未经验证的估算、误用的临界值、单一原因的硬解读
4. 一审集体盲点：所有评审都漏掉的问题（往往是教科书规则被错误套用）

你被允许、并且被鼓励：
- 跑数值（Python/scipy）兜底验证任何定量声明
- 对一审者打过分但你不同意的项发表反对意见
- 提出比一审更尖锐的批评——一审过于宽容是常态
```

## 5. 已知有效的二审产出（worked example）

以下是 Q2 章节二审的实际产出，作为本模板的执行示例：

### 5.1 触发本次二审的一审组合
- **R1** Gemini-style：评分 9.5/10，找到 0 个真错（最宽松）
- **R2** ChatGPT-style：评分 8.5/10，找到 1 真错（NIS 方向）+ 1 实质局限（Bootstrap 参数化）
- **R3** DeepSeek-style：未给评分，找到 1 实质局限（$J^*$ 代数恒等）

三方加总有 4 项独立的"必须修正"，互不重叠——任何单一评审都覆盖不全。

### 5.2 二审在三方一审之外额外找到的问题

| 二审编号 | 问题 | 类别 | 三方为何漏 |
|---|---|---|---|
| #1 | "29σ 越界" 假设 NIS 时序 i.i.d.（实际相关） | D.3 | 三方都接受了 i.i.d. mean 公式 |
| #2 | "$\sigma_a$ 调高 → $P_{xx}$ 升至 0.04–0.06" 是口头估算 | D.1 / F.2 | 三方都没要求数值兜底 |
| #3 | "Bootstrap < CRB 因为乐观偏差" 是单一原因解读 | B.4 | R2 给出该解读，三方未挑战 |
| #4 | overview 表"双重证据"、4.6.1 "对 LS 影响极小" 等 5 处残留旧措辞 | C.1 / C.3 | 一审仅看具体段落，未做跨章节扫查 |
| #5 | "$\chi^2(2)$ 95% 上界 2.99" 在三种 N 语境下是三个不同数 | **E.1（集体盲点）** | 三方均接受 2.99 为权威值 |
| #6 | $J^* = 2(\hat\sigma^2)$ 实为代数恒等（R3 抓到，R1/R2 漏）→ 应进一步在所有相关处下调 | E.2 | R1/R2 仅说"内部自洽"，未识别恒等 |

### 5.3 二审产出的修正性质

- 5 处"修正过弱" → 加强（量化越界、明确恒等式、列举多原因）
- 4 处"跨章节残留" → 全篇 grep 扫查后批量同步
- 1 处"未验证估算" → 跑 Riccati 数值替换口头估算
- 1 处"集体盲点" → 临界值 2.99 → 2.04（i.i.d.）/ 2.46（相关）/ 5.99（单样本）三档分清

总修改量：Q2.md +24 / −20 行（在一审修正基础上）。

## 6. 使用方式

1. **新建审查实例**：复制本模板为 `<章节>_二审清单.md`（如 `Q3_二审清单.md`）。
2. **填充第 0 节"项目背景"** 和**第 1 节"审查输入"**（粘贴一审反馈、补丁 diff、修正决策）。
3. **直接把整份清单贴给 ChatGPT / Gemini / DeepSeek / 另一个 Claude 实例**，要求按第 3 节"输出格式"返回。
4. **建议运行 ≥ 2 个二审实例**——二审本身也可能有盲点。
5. **二审反馈到位后**，作者执行 P0 修正，**必要时再开三审**（极少需要，但学位论文 / 期刊投稿建议至少做到 2.5 审）。

---

## 7. 与一审清单的区别

| 维度 | 一审（红队验证） | 二审（修正审查） |
|---|---|---|
| 输入 | 论文章节 | 论文章节 + 一审反馈 + 修正 diff |
| 任务 | 找到论文的错 | 审查修正本身的错 |
| 关键技巧 | 公式逐条核对、文献对照 | 跨章节 grep、数值兜底、集体盲点扫查 |
| 失败模式 | 漏掉问题 | 把好东西误删 / 假装修了 |
| 性价比 | 高（一审产出大） | 中（多在挑刺细节，但避免 regression） |
| 建议次数 | 至少 2 个独立模型 | 至少 1 个独立模型 |
