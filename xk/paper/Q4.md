# 第六章 问题四：基于附件 3 轨迹的多目标任务调度优化

## 6.1 问题分析与建模思路

附件 3 提供机器人实测运动轨迹（10 Hz 由 Q3 的 KF/RTS 平滑融合给出，含位置、速度、后验方差）。沿途分布 18 个射击目标（S01–S18）与 18 个拍照目标（P01–P18）；在题面附录约束下（距离 / 速度 / 加速度 / 校准时间 / 角度差），需要在轨迹时长内**完成尽可能多的任务**，并按 `result_template.xlsx` 格式输出（序号 / 目标编号 / 任务 / 开始准备时刻 / 任务执行时刻）。

本问题在数学结构上是一个**带连续时间约束的离散组合优化问题**：

| 子问题 | 性质 | 求解策略 |
|---|---|---|
| (a) 候选可行段 | 连续轨迹 + 多重物理约束 + 持续时间窗 | 滑动窗判定 (检查窗内每个采样点) |
| (b) 段内执行时刻 | 连续选择 | 启发式取段距离最小点 |
| (c) 拍照多角度 | 同目标多次去重 | 视角差 ≥ 60° 增量保留 |
| (d) 任务调度 | 离散 0-1 选择 + 时段不冲突 + 覆盖均衡 | 分层目标 ILP (拍照覆盖 ≻ 射击覆盖 ≻ 总数) |
| (e) 鲁棒性 | KF 不确定度对边界候选的风险 | chance constraint + 过渡时间 ε |

**核心方法学贡献**：分治法 — 先把连续时空约束转化为离散候选集，再用 **分层目标 ILP** 同时优化"覆盖目标数"与"总任务数"，破除等权 ILP 在拍照/射击间的平局，并通过**鲁棒约束消除边界候选风险**。

整体建模流程：

```
Q3 KF 轨迹 + 附件4 目标
   │
   ├─→ 滑动窗候选生成 (W=15 射击 / 5 拍照)
   │      ├─→ 标称: d, v, |a| ≤ 阈值
   │      └─→ 鲁棒: d ± z·σ_d, v + z·σ_v ≤ 阈值 (chance constraint, z=1.645)
   │
   ├─→ 段内取 d_min 为执行时刻
   ├─→ 拍照同目标按视角差 ≥ 60° 去重 (题面硬约束)
   │
   └─→ 分层目标 ILP 调度
          ├─ max  M_p · Σy_g_photo  +  M_s · Σy_g_shoot  +  Σ x_i
          │   (M_p = 1000 ≫ M_s = 100 ≫ 1, 拍照覆盖最优先)
          ├─ y_g ≤ Σ_{i: target_i=g} x_i   (覆盖指示)
          ├─ 时段冲突: t_exec_i + ε ≤ t_prep_j
          └─ 射击同目标 ∑ x_k ≤ K = 3 (命中率累积拐点)
```

## 6.2 模型假设

**假设 6.1**（轨迹来源）  机器人位置 / 速度由 Q3 的 KF/RTS 输出给出；位置 1σ 不确定度 σ ≈ 0.60 m，速度 1σ ≈ 5%·|v|（保守估计）。

**假设 6.2**（持续约束的离散化）  题面"校准 / 对准期间约束需持续满足"在 10 Hz 采样下离散化为"窗内全部采样点满足"。由于 KF 信号在 0.1 s 内几乎线性，离散到连续的等价误差 < 1%。

**假设 6.3**（任务互斥）  机器人在同一时刻只能执行一种任务（射击 / 拍照），不允许并发。任务间有 ε = 0.1 s 过渡时间（武器 / 相机模式切换）。

**假设 6.4**（完成数定义）  题面"尽可能多地完成任务"按字面解释为**最大化总任务次数 ∑ x_i**（包括同目标重射）；同时为避免 ILP 因射击候选稠密而挤压拍照槽，引入**分层覆盖优先权重**：先最大化拍照覆盖目标数，其次射击覆盖目标数，最后总任务数。射击同目标允许 K = 3 次（85% → 99.66% 命中率累积拐点）；拍照同目标在视角差 ≥ 60° 下视为独立任务（题面硬约束）。

**假设 6.5**（执行时刻选择启发式）  在窗内全部满足约束的前提下，取**段距离最小点**为执行时刻。对射击：距离越近，机械瞄准误差越小；对拍照：距离越近，成像分辨率越高。

**假设 6.6**（鲁棒性 z 值）  chance constraint 取 z = 1.645（90% 单侧置信），即在边界附近保证 90% 概率不违反约束。

## 6.4 数学模型建立

### 6.4.1 候选可行段（滑动窗判定）

对每个目标 $g$ 与每个时刻 $j$，定义瞬时可行性：

**标称模式**（无 KF 不确定度）：
$$\mathrm{ok}_j^{\mathrm{nom}} = \mathbb 1\{d_j \in [d_{\min}, d_{\max}]\,\wedge\, v_j \le v_{\max}\,\wedge\, |a_j| \le a_{\max}\}. \tag{6.1}$$

**鲁棒模式**（chance constraint, z = 1.645 = 90% 单侧）：
$$\mathrm{ok}_j^{\mathrm{rob}} = \mathbb 1\{d_j - z\sigma_{d,j} \ge d_{\min}\,\wedge\, d_j + z\sigma_{d,j} \le d_{\max}\,\wedge\, v_j + z\sigma_{v,j} \le v_{\max}\,\wedge\, |a_j| \le a_{\max}\}. \tag{6.2}$$

距离不确定度由误差传播：
$$\sigma_{d,j}^2 = \left(\frac{x_j - x_g}{d_j}\right)^2 \sigma_{x,j}^2 + \left(\frac{y_j - y_g}{d_j}\right)^2 \sigma_{y,j}^2. \tag{6.3}$$

**滑动窗判定**：以 $j$ 为准备开始时刻，需要 $[j, j+W)$ 全部满足：
$$\mathrm{OK}_j = \prod_{k=j}^{j+W-1} \mathrm{ok}_k. \tag{6.4}$$

### 6.4.2 段内执行时刻与候选

在连续 $\mathrm{OK}_j = 1$ 段 $[j_0, j_1]$ 内，执行时刻取距离最小：
$$j^*_{\text{exec}} = \arg\min_{k \in [j_0 + W - 1,\ j_1 + W - 1]} d_k,\qquad t_{\text{exec}} = t_{j^*_{\text{exec}}},\quad t_{\text{prep}} = t_{\text{exec}} - T_{\text{win}}. \tag{6.5}$$

每个目标产生一个或多个候选（每段一个）。**拍照同目标多角度去重**：按 $t_{\text{exec}}$ 排序，逐个保留与已选候选视角差 $\ge 60°$ 的：
$$\min_{c' \in \text{已选}} |\theta_g(\mathbf p_{j^*}) - \theta_g(\mathbf p_{j'^*})| \mod 2\pi \ge \frac{\pi}{3}. \tag{6.6}$$

### 6.4.3 分层目标 ILP 调度

引入覆盖指示变量 $y_g \in \{0, 1\}$（$g$ 为 (任务类型, 目标) 对，共 $G = 36$ 个）：

$$
\boxed{
\begin{aligned}
&\max_{\{x_i\},\,\{y_g\}} \quad M_p \!\!\sum_{g \in \mathcal G_{\text{photo}}}\!\! y_g \;+\; M_s \!\!\sum_{g \in \mathcal G_{\text{shoot}}}\!\! y_g \;+\; \sum_{i=1}^n x_i \\[4pt]
&\text{s.t.}\quad y_g \le \sum_{i:\,\text{cand}_i = g} x_i \qquad \forall\,g \in \mathcal G_{\text{photo}} \cup \mathcal G_{\text{shoot}} \quad\text{(覆盖指示)} \\
&\qquad x_i + x_j \le 1 \qquad \forall(i,j):\ t_{\text{exec},i} + \epsilon > t_{\text{prep},j}\,,\ i < j \quad\text{(时段不重叠)} \\
&\qquad \sum_{k:\,\text{cand}_k = (\text{射击},\,g)} x_k \le K \qquad \forall g \in \{S01..S18\},\ K = 3 \quad\text{(射击同目标累积命中拐点)} \\
&\qquad x_i, y_g \in \{0, 1\}.
\end{aligned}
} \tag{6.7}$$

**分层权重**：$M_p = 1000 \gg M_s = 100 \gg 1$。这是字典序优化的近似——先最大化拍照覆盖，再最大化射击覆盖，最后最大化总任务数。该结构在等权目标下会因射击候选稠密（295 个）淹没拍照候选（28 个）—— 等权 ILP 会给出"31 射击 + 4 拍照 = 35"的失衡解，而分层目标给出"29 射击 + 5 拍照 = 34"（拍照达物理上限）的均衡解。

求解：scipy.optimize.milp（HiGHS solver），$n = 323$ 候选 + $G = 36$ 覆盖变量，亚秒级收敛。

### 6.4.4 期望命中数

由射击 85% 命中率，期望击中目标数：
$$E[\text{Hits}] = 0.85 \cdot N_{\text{shoot}}. \tag{6.8}$$

## 6.5 求解算法

```
输入: Q3 KF/RTS 10 Hz 轨迹 (含 X, Y, Vx, Vy, var_X, var_Y)
      附件4 目标 (18 射击 + 18 拍照)
输出: 标称 + 鲁棒双解 (任务序列, result.xlsx 格式)

1. 加载轨迹与目标:
   读取 Q3_trajectory_10Hz_kalman.xlsx → (t, x, y, vx, vy, σ_x, σ_y)
   求加速度: ax, ay = 中心差分(vx, vy)
   求 σ_v = 5% × |v| (Claude 估计, 保守)

2. 候选生成 (双模式):
   对模式 ∈ {标称, 鲁棒}:
     射击候选 ← shoot_candidates(目标, 轨迹, W=15, robust=mode)
     拍照候选 ← photo_candidates(目标, 轨迹, W=5, robust=mode, ang=60°)

3. ILP 调度 (双模式):
   selected_nom ← scipy.milp(候选标称, ε=0)
   selected_rob ← scipy.milp(候选鲁棒, ε=0.1s)

4. 双解对照:
   公共 = selected_nom ∩ selected_rob
   仅标称 = selected_nom - selected_rob (鲁棒下被剔除)
   仅鲁棒 = selected_rob - selected_nom (鲁棒下新增)

5. 输出:
   主交付 = selected_rob (鲁棒解, P0 修正后)
   填入 result_template.xlsx; 保存 Q4_summary.json (双解 + 对照)
```

## 6.6 灵敏度与稳健性分析

### 6.6.1 拍照硬上限的物理证明（核心发现）

按题面 60° 视角差去重后，全部 18 拍照目标共生成 28 个候选；它们的执行时刻**仅集中在 5 个时段**：

| $t_{\text{exec}}$ (s) | 候选数 | 候选目标 |
|---|---|---|
| 445.5 | 14 | P03–P05, P07–P09, P11–P18 |
| 479.9 | 7  | P01, P02, P04, P06, P10, P12, P13 |
| 480.4 | 1  | P14 |
| 493.4 | 4  | P02, P03, P10, P11 |
| 508.3 | 2  | P01, P02 |

由于每个时段内候选互相时段重叠（同一 $t_{\text{exec}} = t_{\text{prep}} + 0.5$ s），**每段最多只能选 1 个**。独立运行 ILP（仅含拍照候选）求拍照覆盖上界：

$$\max_{\{x_i\}} \sum_{i \in \text{photo}} x_i\ \ \text{s.t.}\ \ \text{时段不冲突 + 同目标} \le 1\ \ \Longrightarrow\ \ \boxed{N_{\text{photo}}^{\max} = 5}.$$

**这是题面（视角差 60° + 持续 0.5 s 对准）与轨迹（$|v| \le 1.5$ m/s 的低速段长度）的乘积所决定的物理硬上限**——不是模型缺陷或求解器局限。机器人在 $[445, 510]$ s 的低速段中只产生 5 个互不冲突的拍照机会窗。

### 6.6.2 等权 ILP 的失衡问题与分层目标修复

**症状**：等权目标 $\max \sum x_i$（无覆盖权重）下 ILP 给出 **31 射击 + 4 拍照 = 35**——拍照只覆盖 4 个目标，远低于物理上限 5。

**诊断**：射击候选 295 个、拍照候选仅 28 个；同时段内 ILP 平局时倾向稠密放置射击（K = 3 允许同目标多次）。增加 1 个拍照覆盖 = 失去 1 个射击覆盖、净增益 = 0，ILP 无动力做"覆盖均衡"。

**修复**：分层权重 $M_p = 1000 \gg M_s = 100 \gg 1$（式 6.7）。引入覆盖指示变量 $y_g$ 后，每多覆盖 1 个拍照目标增益 $+1000$，远超失去的射击覆盖收益 $-100$，强制 ILP 优先填满 5 个拍照槽。

**修复效果**：

| 目标函数 | 总数 | 射击 | 拍照 | 拍照覆盖 | 射击覆盖 | 是否达拍照上限 |
|---|---|---|---|---|---|---|
| 等权 $\max \sum x_i$ | 35 | 31 | 4 | **4 / 18** | 14 / 18 | ✗ |
| 分层（**主解**） | **34** | 29 | **5** | **5 / 18** ★ | 15 / 18 | ✓ |

主解仅减 1 任务（35 → 34）即换得拍照覆盖从 4 升到 5（达物理上限），是**用最小代价达到题面下最优均衡**的代表方案。

### 6.6.3 三审 P0 修正：chance constraint 鲁棒约束

经 DeepSeek + Claude + Gemini 三方红队审查（详见 `Q4_三审综合.md`），原标称解中部分候选位于约束边界，KF 不确定度下违反约束概率高。引入 **chance constraint**（$z = 1.645$，等价 90% 单侧置信）：

- $d - 1.645\,\sigma_d \ge 5$（射击下界）⇒ 实际 $d \ge 5 + 1.645 \times 0.6 \approx 5.99$ m
- $v + 1.645\,\sigma_v \le 1.5$ ⇒ 等价于把 $v_{\max}$ 收紧约 5%

### 6.6.6 加速度估计的诚实定性

加速度由 KF 速度的中心差分得到：
$$\sigma_a \approx \sqrt{2}\sigma_v / \mathrm{dt} \approx \sqrt{2} \times 0.05 |v| / 0.1 \approx 0.7 |v|.$$

在 $|v| \approx 2$ m/s 时 $\sigma_a \approx 1.4$ m/s²，**与真实加速度量级相近**——估计存在显著噪声。**但实证：入选任务 $|a| \le 0.847$ 远小于约束 $1.5$，加速度约束在最优解处不主动起作用**。该局限不影响主解，作论文 §7 已知缺陷记录。

### 6.6.7 时间分布与覆盖率的物理解释

入选 34 任务中，**21 个集中在 $[445, 510]$ s**（65 s 内）——对应 KF 速度 $\le 1.5$ m/s 的低速段（拍照可达）；**13 个分布在 $[510, 808]$ s**——主要为高速段中的射击补射（射击 $v \le 2$ 限制比拍照宽松 33%）。

**覆盖率不对称的物理原因**：

| 任务类 | 入选目标覆盖 | $d$ 范围 | 物理原因 |
|---|---|---|---|
| 射击 | **15 / 18 = 83%** | $[5, 30]$ m | 速度上限宽松，沿轨迹多段可达；S02、S17、S18 在 $Y > 50$ 不可达 |
| 拍照 | **5 / 18 = 28%** | $[10, 40]$ m | 速度上限严苛，**仅低速段 [445, 510] s 的 5 个互不冲突槽** |

拍照覆盖低不是模型缺陷——是题面"$v \le 1.5$ + 视角差 60° + 持续 0.5 s"三重约束与机器人轨迹（多为高速段）的乘积。机器人在 $[510, 800]$ s 高速段每个时刻都不满足 $v \le 1.5$，无法拍照。要提升拍照覆盖必须修改轨迹（轨迹—任务联合规划），不属于本问题范畴。

### 6.7.3 物理解释

- **阶段 II [533, 808.8] s（276 s 中 13 任务）**：13 射击（S10×3, S11×3, S12×3, S13–S16 各 1 次）。该段速度多为 $1.0$–$2.0$ m/s，超过拍照阈值 1.5 但满足射击阈值 2.0；S10、S11、S12 三个目标距离适中（$\sim 25$ m）支持 K=3 重射。

## 6.8 优缺点分析

**优点**：

1. **分治法处理时空约束**：将连续轨迹上的物理约束（d, v, |a|, 持续时间）转化为离散候选集（候选生成阶段），再用 ILP 求全局最优——是处理"带连续约束的组合优化"的标准高效范式。
2. **分层目标 ILP 解决覆盖失衡**（**核心方法学贡献**）：等权 $\max \sum x_i$ 因射击候选稠密（295 vs 28）会给出"31 射击 + 4 拍照"失衡解；引入分层权重 $M_p \gg M_s \gg 1$ 强制拍照覆盖优先，仅以 1 任务代价换得拍照覆盖从 4 → 5（达物理上限）。
3. **拍照硬上限的物理证明**：独立 ILP 验证拍照在题面约束下覆盖上限为 5；该结论将"覆盖失衡"从模型缺陷澄清为题面与轨迹的物理后果。
4. **ILP 全局最优**：scipy.milp + HiGHS solver 在 $n = 323$ 候选 + 36 覆盖变量规模下亚秒级收敛，**保证理论最优**（不是贪心近似）。
5. **chance constraint 鲁棒化**：通过 $d - z\sigma_d \ge d_{\min}$ 等保守约束消除边界候选风险，给出"标称（题面字面）+ 鲁棒（90% 置信工程）"双解对照。
6. **K=3 命中率工程拐点**：3 次累积 99.66%、4 次仅 +0.29% 边际收益，K=3 是综合"尽可能多任务"与"避免重复浪费"的平衡点。
7. **诚实揭示局限**：加速度估计噪声、拍照硬上限、覆盖率不对称的物理原因均在论文中明确记录。

**缺点与局限**：

1. **"段内取 d_min"是启发式**：在保证窗内全程满足约束的前提下，取 $d$ 最小并非严格最优；但在 KF 噪声背景下，距离最近能最大化 SNR，工程上合理。
2. **加速度估计噪声**：中心差分使 $\sigma_a$ 与真实 $|a|$ 同量级；但**约束不主动起作用**（入选 $|a| \le 0.847 \ll 1.5$），影响有限。
3. **拍照覆盖受轨迹限制**：5 / 18 拍照覆盖是题面 + 轨迹乘积的物理硬上限；要再提升只能修改轨迹（轨迹—任务联合规划），属于"问题外延"。
4. **K=3 选择需进一步实证**：K 是"尽可能多任务"语义化展开的解释参数；可能存在评委倾向 K=1（严格唯一）的解读分歧——故附 K 扫描表（§6.6.5）供评委选择层级。
5. **未考虑相机/武器系统并发**：保守假设单任务串行；如果工程上允许并发，可放松互斥约束。

## 6.9 本节小结

本章基于附件 3 实测轨迹（来自 Q3 的 KF/RTS 平滑融合），建立了**滑动窗候选生成 + 视角去重 + 分层目标 ILP**的多目标任务调度优化框架：

- **主解（标称 + 覆盖均衡 ILP）**：**34 任务 = 29 射击 + 5 拍照**，时间跨度 $[445, 808.8]$ s，期望击中目标 24.65；覆盖 15 射击 + 5 拍照 = 20 个不同 (任务 × 目标) 对（55.6%）。
- **鲁棒解（chance constraint, z=1.645）**：31 任务 = 27 射击 + 4 拍照，所有约束 90% 置信下严格满足，作为主解的工程鲁棒对照。
- **拍照硬上限发现**：独立 ILP 验证 $N_{\text{photo}}^{\max} = 5$——题面"$v \le 1.5$ + 视角差 60° + 持续 0.5 s"与机器人轨迹（仅 $[445, 510]$ s 为低速段）的乘积所决定的物理上限；主解已达到该上限。
- **分层目标 ILP 必要性**：等权 $\max \sum x_i$ 因射击候选 295 vs 拍照候选 28 给出"31 射击 + 4 拍照"失衡解；分层权重 $M_p = 1000 \gg M_s = 100 \gg 1$ 强制拍照覆盖优先，**仅减 1 任务即换得拍照覆盖从 4 → 5（物理上限）**。
- **K=3 工程拐点**：单次命中 0.85，3 次累积 0.9966；$K \ge 4$ 边际收益 < 0.3% 不足以补偿 1.5 s 时段成本。
- **覆盖率不对称的物理解释**：拍照 5/18 不是模型缺陷，而是题面约束 + 轨迹高速段比例决定的硬上限；要再提升只能修改轨迹（轨迹—任务联合规划）。

本章建立的 **滑动窗 + 分层目标 ILP + chance constraint** 范式适合所有"带连续约束 + 多类型任务覆盖均衡"的离散调度问题。在保证 30+ 任务、严格守题面、拍照达物理上限三大目标下给出最优解。
